## 1. 시스템 개요

HMS(Hospital Management System)는 병원의 예약·접수·진료·물품 관리를 통합하는 웹 기반 시스템입니다.

### 1.1 주요 특징

- **AI 증상 분석 예약** — 환자가 증상을 입력하면 AI가 적합한 진료과·의사를 자동 추천
- **AI 의료 상담** — 의학 지식 기반 AI 챗봇으로 의료 질문 답변
- **역할 기반 접근 제어** — 5개 역할(ADMIN, DOCTOR, NURSE, STAFF, ITEM_MANAGER)별 전용 화면
- **실시간 예약 현황** — 접수·진료·수납 상태를 실시간으로 추적
- **병원 규칙 Q&A 챗봇** — 내부 직원용 병원 규정·절차 AI 챗봇

### 1.2 예약 상태 흐름

```
[예약(RESERVED)] → [접수/진료대기(RECEIVED)] → [진료중(IN_TREATMENT)] → [진료완료(COMPLETED)] → [처치완료] → [수납완료]
                                                                                ↑
[취소(CANCELLED)] ←──────────────────────────────────────────────────────────────┘ (언제든 취소 가능)
```

| 상태 | 설명 |
|------|------|
| `RESERVED` (예약) | 온라인/전화 예약이 등록된 상태 |
| `RECEIVED` (접수/진료대기) | 원무과에서 접수 완료, 진료 대기 중 |
| `IN_TREATMENT` (진료중) | 의사가 진료를 시작한 상태 |
| `COMPLETED` (진료완료) | 의사가 진료 기록을 작성하고 완료 처리 |
| 처치완료 | 간호사가 처치를 완료한 상태 (수납 가능) |
| 수납완료 | 접수 직원이 수납 처리를 완료한 상태 |
| `CANCELLED` (취소) | 예약이 취소된 상태 |

### 1.3 예약 경로 (Source)

| 경로 | 설명 |
|------|------|
| `ONLINE` | 환자가 웹사이트에서 직접 예약 |
| `PHONE` | 접수 직원이 전화를 받아 대리 예약 |
| `WALKIN` | 환자가 병원에 직접 방문하여 즉시 접수 |

## 3. 공통 기능

### 3.1 로그인

1. 브라우저에서 `https://<서버IP>/login` 접속
2. **사용자명**(username)과 **비밀번호**(password) 입력
3. **로그인** 버튼 클릭
4. 역할에 따라 해당 대시보드로 자동 이동

> **참고:** 동일 계정으로 동시 로그인 시 이전 세션은 자동 만료됩니다 (동시 세션 1개 제한).

### 3.2 로그아웃

- 사이드바 하단 또는 헤더의 **로그아웃** 버튼 클릭
- 세션이 즉시 무효화되며 로그인 페이지로 이동

### 3.3 마이페이지

모든 내부 직원 역할에 마이페이지가 제공됩니다.

- **접속 경로:** 사이드바의 **마이페이지** 메뉴
- **주요 기능:** 내 정보 확인, 비밀번호 변경

### 3.4 AI 챗봇

로그인한 모든 직원은 범용 AI 챗봇을 사용할 수 있습니다.

- **접속 경로:** 사이드바의 **AI 챗봇** 메뉴
- **기능:** 자유로운 질문·답변 (일반 대화, 업무 관련 질문 등)

## 4. 비회원 — 외부 예약 시스템

비회원(환자)은 로그인 없이 예약 관련 기능을 이용할 수 있습니다.

### 4.1 메인 화면

- **접속:** `https://<서버IP>/`
- 메인 페이지에서 **진료 예약** 또는 **예약 조회** 선택

### 4.2 예약 방식 선택

- **접속:** `/reservation`
- 두 가지 예약 방식 중 선택:
  - **AI 증상 분석 예약** — 증상을 입력하면 AI가 진료과·의사를 추천
  - **직접 선택 예약** — 진료과·의사·날짜·시간을 직접 선택

### 4.3 AI 증상 분석 예약

1. **접속:** `/reservation/symptom-reservation`
2. 증상을 자연어로 입력 (예: "머리가 아프고 열이 나요")
3. AI가 증상을 분석하여 적합한 **진료과**와 **의사**를 추천
4. 추천 결과를 확인하고 **날짜**와 **시간** 선택
5. **환자 정보** 입력 (이름, 전화번호)
6. **예약 완료** → 예약번호 발급

### 4.4 직접 선택 예약

1. **접속:** `/reservation/direct-reservation`
2. **진료과** 선택 (내과, 외과, 소아과, 이비인후과 등)
3. 선택한 진료과의 **의사** 목록에서 선택
4. **예약 날짜**와 **시간대** 선택
5. **환자 정보** 입력 (이름, 전화번호)
6. **예약하기** 버튼 클릭
7. **예약 완료 화면** — 예약번호, 환자명, 진료과, 의사, 날짜, 시간 확인

> **주의:** 같은 의사에게 같은 날짜·시간에 중복 예약은 불가합니다.

### 4.5 예약 조회

1. **접속:** `/reservation/lookup`
2. 조회 방법:
   - **예약번호로 조회** — 예약번호 입력 후 검색
   - **이름 + 전화번호로 조회** — 환자명과 전화번호 입력 후 검색
3. 검색 결과에서 예약 상세 정보 확인

### 4.6 예약 변경

1. **접속:** `/reservation/modify`
2. **예약번호** 입력 후 조회
3. 예약 정보 확인 후 **날짜**, **시간** 등 변경 사항 입력
4. **전화번호** 입력 (본인 확인용)
5. **변경하기** 버튼 클릭
6. **변경 완료 화면** 확인

### 4.7 예약 취소

1. **접속:** `/reservation/cancel`
2. **예약번호** 입력 후 조회
3. 예약 정보 확인
4. **전화번호** 입력 (본인 확인용)
5. **취소하기** 버튼 클릭
6. **취소 완료 화면** 확인

### 4.8 AI 의료 상담

1. **접속:** `/llm/medical`
2. 의료 관련 질문을 자유롭게 입력
3. AI가 의학 지식 기반으로 답변 제공

> **참고:** AI 의료 상담은 참고 목적이며, 정확한 진단은 반드시 의사의 진료를 통해 받으세요.

## 5. 접수 직원 (STAFF)

접수 직원은 환자의 예약 접수, 전화 예약, 방문 접수, 수납을 담당합니다.

### 5.1 접수 대시보드

- **접속:** `/staff/dashboard`
- **표시 정보:** 오늘 예약 건수, 접수 현황, 대기 환자 수 등 요약 통계

### 5.2 접수 목록

- **접속:** `/staff/reception/list`

#### 탭 구분

| 탭 | 설명 |
|----|------|
| **전체** | 오늘의 모든 예약 표시 |
| **예약** | `RESERVED` 상태 — 접수 대기 중인 예약 |
| **접수** | `RECEIVED` 상태 — 접수 완료, 진료 대기 중 |
| **진료현황** | 진료중·진료완료·처치완료 환자 표시 |
| **수납** | 처치 완료(수납 대기) + 수납 완료 환자 |
| **취소** | 취소된 예약 표시 |

#### 검색·필터 기능

- **날짜 선택** — 특정 날짜의 예약 목록 조회
- **검색어 입력** — 환자명, 예약번호 등으로 검색
- **진료과 필터** — 특정 진료과만 필터링
- **담당의 필터** — 특정 의사만 필터링
- **예약 경로 필터** — 온라인/전화/방문 필터링

#### 페이징

- 10건 단위로 페이지 분할
- 하단 페이지 네비게이션으로 이동

### 5.3 접수 상세

- **접속:** `/staff/reception/detail?id={예약ID}`
- **표시 정보:** 예약 정보, 환자 정보, 진료과, 담당의, 예약 경로 등

#### 주요 작업

| 작업 | 설명 |
|------|------|
| **접수 처리** | `RESERVED` → `RECEIVED` 상태 변경 (환자 도착 시) |
| **예약 취소** | 예약 취소 처리 (사유 입력 가능) |
| **수납 처리** | 처치 완료된 환자의 수납 완료 처리 |
| **환자 정보 수정** | 환자 전화번호 등 기본 정보 수정 |

### 5.4 전화 예약 등록

1. **접속:** `/staff/reservation/phone-reservation`
2. 환자 정보 입력:
   - **환자명**
   - **전화번호**
3. 예약 정보 입력:
   - **진료과** 선택
   - **담당의** 선택
   - **예약 날짜** 선택
   - **시간대** 선택
4. **예약 등록** 버튼 클릭
5. 완료 시 해당 날짜의 접수 목록으로 이동

> **참고:** 기존 전화번호가 등록된 환자인 경우, 기존 환자 정보에 자동으로 연결됩니다. 이름이 다를 경우 안내 메시지가 표시됩니다.

### 5.5 방문 접수

1. **접속:** `/staff/walkin-reception`
2. 환자 정보 입력:
   - **환자명**
   - **전화번호** (010-XXXX-XXXX 형식)
3. 진료 정보 입력:
   - **진료과** 선택
   - **담당의** 선택
4. **접수** 버튼 클릭
5. 즉시 `RECEIVED` (진료대기) 상태로 등록

> **참고:** 접수 목록에서 기존 예약 환자를 선택하여 방문 접수 화면으로 이동하면 정보가 자동으로 채워집니다.

### 5.6 물품 출고 (접수 직원용)

- **접속:** `/staff/item-use`
- 접수 직원도 물품 출고가 가능합니다 (예: 소모품 지급)

## 6. 의사 (DOCTOR)

의사는 진료 목록 확인, 진료 기록 작성, 물품 사용을 담당합니다.

### 6.1 의사 대시보드

- **접속:** `/doctor/dashboard`
- **표시 정보:** 오늘 진료 예정 환자 수, 진료 완료 수, 대기 환자 수 등

### 6.2 진료 현황

- **접속:** `/doctor/treatment-list`

#### 탭 구분

| 탭 | 설명 |
|----|------|
| **대기** | 접수 완료, 진료 대기 중인 환자 |
| **완료** | 진료 완료된 환자 |
| **전체** | 오늘 모든 환자 |

#### 주요 기능

- **날짜 선택** — 특정 날짜의 진료 목록 조회
- **검색** — 환자명으로 검색
- **5초 자동 갱신** — 새로운 접수 환자가 실시간으로 목록에 추가
- **페이징** — 페이지 단위 목록 탐색

### 6.3 진료 상세 (진료실)

1. 진료 목록에서 환자를 클릭하여 **진료 상세** 화면 진입
   - **접속:** `/doctor/treatment-detail?id={예약ID}`

2. **환자 정보 확인:**
   - 환자명, 예약 정보, 과거 진료 이력

3. **진료 시작:**
   - **진료 시작** 버튼 클릭 → 상태가 `IN_TREATMENT`(진료중)로 변경

4. **진료 기록 작성:**
   - **진단명** 입력 (예: "급성 위염")
   - **처방** 입력 (예: "큐란 1정, 가스모틴 1정")
   - **비고** 입력 (선택, 예: "안정 필요")

5. **물품 사용:** (진료 중 소모품 사용 시)
   - 물품 목록에서 사용할 물품 선택
   - 수량 입력 후 **사용** 클릭
   - 사용 취소도 가능 (사용 기록에서 **취소** 클릭)

6. **진료 완료:**
   - **임시 저장** — 진료 기록을 저장하되 상태 유지
   - **진료 완료** — 진료 기록 저장 + 상태를 `COMPLETED`로 변경

### 6.4 진료 완료 목록

- **접속:** `/doctor/completed-list`
- 날짜별 진료 완료 환자 목록 조회
- 진료 차트(진단명, 처방, 비고, 물품 사용 내역) 재확인 가능

### 6.5 병원 규칙 Q&A 챗봇

- **접속:** 사이드바의 **병원 규칙 Q&A** 메뉴
- 병원 내부 규정, 절차, 물품 위치 등을 AI에게 질문
- 등록된 병원 규칙 데이터를 기반으로 답변 제공

## 7. 간호사 (NURSE)

간호사는 환자 현황 조회, 처치 관리, 환자 정보 수정을 담당합니다.

### 7.1 간호사 대시보드

- **접속:** `/nurse/dashboard`
- **표시 정보:** 오늘 환자 현황, 접수 대기, 진료중, 처치 대기 수 등

### 7.2 환자 현황

- **접속:** `/nurse/reception-list`

#### 탭 구분

| 탭 | 설명 |
|----|------|
| **전체** | 오늘 전체 환자 |
| **예약** | 예약 상태 환자 |
| **접수** | 접수 완료, 진료 대기 중 |
| **진료중** | 현재 진료 중인 환자 |
| **진료완료** | 진료 완료, 처치 대기 |
| **처치완료** | 처치까지 완료된 환자 |

#### 주요 기능

- **날짜 선택**, **검색**, **진료과/담당의 필터**
- **접수 처리** — 예약 상태 환자를 접수 처리 가능
- **세션 필터 유지** — 필터 조건이 세션에 저장되어 페이지 이동 후에도 유지

### 7.3 환자 상세

1. 환자 현황 또는 처치 관리 목록에서 환자 클릭
   - **접속:** `/nurse/patient-detail?id={예약ID}`

2. **확인 가능 정보:**
   - 환자 기본 정보 (이름, 전화번호, 주소)
   - 예약/접수 정보
   - 진료 기록 (진단명, 처방, 의사 비고)
   - 물품 사용 내역

3. **환자 정보 수정:**
   - 전화번호, 주소, 메모 수정 후 **저장**

4. **간호 메모 작성:**
   - 간호 관련 메모 입력 후 **저장**

5. **물품 사용:**
   - 처치에 필요한 물품 선택·수량 입력 후 **사용**
   - 사용 취소 가능

### 7.4 처치 관리

- **접속:** `/nurse/treatment-list`

#### 탭 구분

| 탭 | 설명 |
|----|------|
| **처치 대기** | 진료 완료 후 처치가 필요한 환자 |
| **처치 완료** | 처치가 완료된 환자 |
| **전체** | 모든 진료 완료 환자 |

#### 처치 완료 처리

1. 처치 대기 목록에서 환자 선택
2. 필요한 처치 수행 후 **처치 완료** 버튼 클릭
3. 상태가 "처치 완료"로 변경 → 접수 직원의 수납 대기 목록에 표시

### 7.5 병원 규칙 Q&A 챗봇

- 의사와 동일하게 병원 규칙 Q&A 챗봇 사용 가능
- **접속:** 사이드바의 **병원 규칙 Q&A** 메뉴

## 8. 물품 관리자 (ITEM_MANAGER)

물품 관리자는 병원 물품의 등록, 재고 관리, 입출고 내역을 전담합니다.

### 8.1 물품 대시보드

- **접속:** `/item-manager/dashboard`
- **표시 정보:** 전체 물품 수, 재고 부족 물품, 오늘 입출고 요약 등

### 8.2 물품 목록

- **접속:** `/item-manager/item-list`
- 등록된 모든 물품의 목록 표시
- **카테고리 필터:**

| 카테고리 | 설명 |
|----------|------|
| `MEDICAL_SUPPLIES` | 의료 소모품 (거즈, 주사기, 붕대 등) |
| `MEDICAL_EQUIPMENT` | 의료 장비 (혈압계, 체온계 등) |
| `GENERAL_SUPPLIES` | 일반 소모품 (사무용품 등) |

### 8.3 물품 등록/수정

1. **등록:** `/item-manager/item-form` → **물품 등록** 버튼
2. **수정:** 물품 목록에서 수정할 물품 클릭 → `/item-manager/item-form?id={물품ID}`
3. 입력 항목:
   - **물품명**
   - **카테고리** 선택
   - **재고 수량**
   - **최소 수량** (이 이하로 떨어지면 부족 알림)
4. **저장** 버튼 클릭

### 8.4 물품 삭제

- 물품 목록 또는 수정 화면에서 **삭제** 버튼 클릭

### 8.5 물품 입고 (재고 추가)

- 물품 대시보드 또는 목록에서 **입고** 버튼 클릭
- 입고 수량 입력 후 확인
- 현재 재고에 입고 수량이 더해짐

### 8.6 물품 출고

- **접속:** `/item-manager/item-use`
- 출고할 물품 선택 → 수량 입력 → **출고** 클릭
- 당일 출고 기록이 하단에 실시간 표시

### 8.7 입출고 내역

- **접속:** `/item-manager/item-history`
- **기간 검색:** 시작일·종료일 입력하여 기간별 내역 조회
- **표시 정보:** 날짜, 물품명, 입고/출고 구분, 수량, 총 입고량, 총 출고량

## 10. AI 기능 안내

HMS는 두 가지 AI 엔진을 활용합니다.

### 10.1 AI 엔진 구성

| 엔진 | 용도 | 기술 |
|------|------|------|
| **Python LLM 서버** | 증상 분석, 의료 상담, 병원 규칙 Q&A | FastAPI + vLLM (Qwen 2.5-3B) + ChromaDB |
| **Claude API** | 범용 AI 챗봇, 보조 추론 | Anthropic Claude Sonnet |

### 10.2 AI 증상 분석 (비회원)

- 환자가 자연어로 증상을 설명하면 AI가 분석
- 적합한 진료과와 의사를 자동 추천
- 추천 결과를 바탕으로 바로 예약 가능

### 10.3 AI 의료 상담 (비회원)

- 일반적인 의료 질문에 대해 AI가 답변
- 의학 데이터베이스와 벡터 검색(ChromaDB)을 활용한 RAG 기반 응답
- **주의:** 진단 목적이 아닌 참고 정보 제공용

### 10.4 병원 규칙 Q&A 챗봇 (의사·간호사)

- 병원 내부 규정, 절차, 물품 위치 등을 질문
- 관리자가 등록한 병원 규칙 데이터를 기반으로 정확한 답변 제공
- 벡터 검색으로 관련 규칙을 찾아 맥락에 맞는 답변 생성

### 10.5 범용 AI 챗봇 (전 직원)

- 로그인한 모든 직원이 사용 가능
- 업무 관련 질문, 일반 질문 등 자유롭게 대화

### 10.6 AI 장애 시 동작

- LLM 서버 또는 Claude API 장애 시 **폴백(fallback)** 로직이 동작합니다
- 증상 분석 예약의 경우 → **직접 선택 예약** 화면으로 자동 전환
- 챗봇의 경우 → 오류 메시지 표시

## 11. 자주 묻는 질문 (FAQ)

### 일반

**Q: 비밀번호를 잊어버렸습니다.**
A: 관리자(ADMIN)에게 비밀번호 초기화를 요청하세요. 관리자는 직원 관리 화면에서 비밀번호를 재설정할 수 있습니다.

**Q: 다른 기기에서 로그인하면 기존 세션이 끊기나요?**
A: 네, 동일 계정의 동시 세션은 1개로 제한됩니다. 새로운 기기에서 로그인하면 이전 세션은 자동 만료됩니다.

**Q: 세션이 자동으로 끊어집니다.**
A: 운영 환경 기준 세션 타임아웃은 30분입니다. 30분간 활동이 없으면 자동 로그아웃됩니다. 다시 로그인해 주세요.

### 예약

**Q: 환자가 같은 시간에 두 명의 의사에게 예약할 수 있나요?**
A: 같은 의사에게 동일 시간대 중복 예약은 불가능하지만, 다른 의사에게는 예약할 수 있습니다.

**Q: 예약 취소 후 다시 예약할 수 있나요?**
A: 네, 취소된 예약은 복구할 수 없지만, 새로운 예약을 등록할 수 있습니다.

**Q: 환자가 예약 없이 방문했습니다.**
A: 접수 직원이 **방문 접수** 기능(`/staff/walkin-reception`)으로 즉시 접수할 수 있습니다. 방문 접수는 바로 `RECEIVED`(진료대기) 상태로 등록됩니다.

### 진료

**Q: 진료 기록을 수정할 수 있나요?**
A: 진료 완료 전에는 **임시 저장**으로 기록을 수정할 수 있습니다. 진료 완료 후에는 진료 완료 목록에서 기록을 확인할 수 있습니다.

**Q: 진료 중에 물품을 잘못 사용 처리했습니다.**
A: 진료 상세 화면의 물품 사용 내역에서 **취소** 버튼을 클릭하면 사용이 취소되고 재고가 복구됩니다.

### 물품

**Q: 재고 부족 알림은 어디서 확인하나요?**
A: 물품 관리자 대시보드에서 최소 수량 이하로 떨어진 물품이 표시됩니다.

**Q: 물품을 잘못 출고했습니다.**
A: 물품 관리자에게 입고 처리를 요청하여 재고를 보정할 수 있습니다.

### AI

**Q: AI 증상 분석이 작동하지 않습니다.**
A: LLM 서버 장애 시 자동으로 **직접 선택 예약** 화면으로 전환됩니다. 진료과와 의사를 직접 선택하여 예약하세요.

**Q: 병원 규칙 챗봇의 답변이 정확하지 않습니다.**
A: 관리자에게 병원 규칙 데이터 업데이트를 요청하세요. 등록된 규칙이 많을수록 답변 정확도가 향상됩니다.

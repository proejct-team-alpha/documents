# 🏥 병원 예약 & 내부 업무 시스템 — 전체 화면 목록 및 화면별 기능 정의서

> **문서 버전:** v4.0
> **작성일:** 2026년
> **변경 이력:** v1.0 (22개) → v1.1 (29개) → v1.2 (33개) → v1.3 (문서 연동) → v1.4 (공통 레이아웃·헤더·사이드바·푸터 컴포넌트 정의) → v2.0 (역할 분리 및 정보관리 화면 추가) → v3.0 (ROLE_ITEM_MANAGER 폐기 및 API spec v3.0 동기화: 총 33개 + C1) → **v4.0 (진료과 activate/deactivate API 정합화, ITEM.category 한글 레이블 통일, API 명세서 v4.0 연동 반영)**
>
> **v4.0 주요 변경사항:**
>
> - **진료과 activate/deactivate API 정합화** — 활성화: `POST /admin/department/activate`, 비활성화: `POST /admin/department/deactivate` 별도 엔드포인트 사용
> - **ITEM.category 한글 레이블 통일** — 의료용품→의료 소모품, 의료기기→의료 장비, 소모품→일반 소모품 (ERD v3.0 ENUM 정합)
> - **연관 문서 버전 갱신** — API 명세서 v3.0→v4.0, ERD v2.0→v3.0
>   **연관 문서:** 프로젝트 계획서 v4.2 / ERD v3.0 / API 명세서 v4.0 / 시퀀스 다이어그램 v3.0
>   **렌더링 방식:** Spring Boot + Mustache SSR (PRG 패턴)
>   **인증 방식:** 세션 기반 (Spring Security)

## 2. 공통 레이아웃 컴포넌트

---

### 2.1 레이아웃 유형 정의

모든 화면은 아래 세 가지 레이아웃 중 하나를 사용합니다.
각 화면의 `적용 레이아웃` 항목에서 어떤 레이아웃이 적용되는지 확인합니다.

| 코드   | 레이아웃 이름        | 헤더 | 사이드바       | 푸터 | 적용 화면                     |
| ------ | -------------------- | ---- | -------------- | ---- | ----------------------------- |
| **L1** | 비회원 공개 레이아웃 | H1   | 없음           | F1   | 화면 00~04 (비회원 예약 흐름) |
| **L2** | 내부 직원 레이아웃   | H2   | S1~S4 (ROLE별) | F2   | 화면 06~32 (로그인 직원)      |
| **L3** | 로그인 전용 레이아웃 | H3   | 없음           | 없음 | 화면 05 (로그인)              |

---

### 2.2 헤더 컴포넌트

#### H1 — 비회원 헤더

> **Mustache 파셜:** `{{> common/header-public}}`
> **적용 화면:** L1 레이아웃 전체 (화면 00~04)

```
┌─────────────────────────────────────────────────────────────────┐
│  🏥 ○○병원                                       [직원 로그인]   │
└─────────────────────────────────────────────────────────────────┘
```

| 요소               | 설명                           |
| ------------------ | ------------------------------ |
| 병원 로고·이름     | 클릭 시 `/` 이동               |
| [직원 로그인] 버튼 | `/login` 이동 (헤더 우측 고정) |

---

#### H2 — 내부 직원 헤더

> **Mustache 파셜:** `{{> common/header-staff}}`
> **적용 화면:** L2 레이아웃 전체 (화면 06~32)

```
┌─────────────────────────────────────────────────────────────────┐
│  🏥 ○○병원 내부 시스템   [현재 메뉴명]   홍담당 (STAFF) [로그아웃] │
└─────────────────────────────────────────────────────────────────┘
```

| 요소               | 설명                                                  |
| ------------------ | ----------------------------------------------------- |
| 병원 로고·이름     | 클릭 시 ROLE별 대시보드 이동                          |
| 현재 메뉴명        | 현재 화면 이름 표시 (Mustache 모델 전달)              |
| 로그인 이름 + ROLE | 세션 사용자 이름 + `(STAFF / DOCTOR / NURSE / ADMIN)` |
| [로그아웃] 버튼    | `POST /logout` 처리                                   |

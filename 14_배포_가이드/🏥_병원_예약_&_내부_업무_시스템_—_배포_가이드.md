## 1. 배포 환경

### 1.1 권장 사양

### 1.2 시스템 아키텍처

```
[Client Browser]
       |
       | HTTPS (443)
       v
  [Nginx Container]  ── hms-nginx
       |
       | HTTP (8080)
       v
  [Spring Boot Container]  ── hms-spring (JDK 21 + Spring Boot 4.0.3)
       |
       +──→ [MySQL Container]       ── hms-db       (3306)
       +──→ [Redis Container]       ── hms-redis    (6379)
       +──→ [Python LLM Container]  ── hms-python   (8000)
       |         |
       |         +──→ [vLLM / Ollama Server]  (외부: 9000/11434)
       |         +──→ [ChromaDB Container]    ── hms-chromadb (8100→8000)
       |         +──→ [MySQL Container]       ── hms-db (의학 데이터 조회)
       |
       +──→ [Claude API]  (HTTPS 외부 — 선택)
```

### 1.3 컨테이너 구성

| 컨테이너       | 이미지                  | 포트 (호스트:컨테이너) | 설명                                  |
| -------------- | ----------------------- | ---------------------- | ------------------------------------- |
| `hms-nginx`    | `nginx:alpine`          | `443:443`, `80:80`     | 리버스 프록시, SSL 종단               |
| `hms-spring`   | 멀티 스테이지 빌드      | 내부 `8080` (expose)   | Spring Boot 메인 애플리케이션         |
| `hms-python`   | `python:3.11-slim`      | `8000:8000`            | LLM 추론 서버 (FastAPI + vLLM/Ollama) |
| `hms-db`       | `mysql:8.0`             | `3306:3306`            | 메인 데이터베이스                     |
| `hms-redis`    | `redis:7-alpine`        | `6379:6379`            | 세션 저장·캐시                        |

## 2. 사전 준비

### 2.1 Docker & Docker Compose

```bash
# Ubuntu 22.04
sudo apt update
sudo apt install -y docker.io docker-compose-v2

# Docker 서비스 시작
sudo systemctl start docker
sudo systemctl enable docker

# 현재 사용자를 docker 그룹에 추가 (재로그인 필요)
sudo usermod -aG docker $USER

# 설치 확인
docker --version
docker compose version
```

### 2.2 JDK 21 (로컬 개발 / 네이티브 빌드 시)

```bash
# Ubuntu 22.04
sudo apt install -y openjdk-21-jdk

# 설치 확인
java -version
# openjdk version "21.x.x"
```

### 2.3 Node.js 20+ (프론트엔드 빌드)

> Docker 빌드 시 자동으로 `node:20-slim` 스테이지에서 처리되므로 Docker 배포 시 별도 설치 불필요.

```bash
# 로컬 개발 시 필요
sudo apt install -y nodejs npm
node -v  # v20+

# 의존성 설치
npm ci
```

### 2.4 Python 3.11+ (LLM 서버 로컬 실행 시)

> Docker Compose 배포 시 `python:3.11-slim` 이미지로 자동 빌드되므로 별도 설치 불필요.

```bash
# 로컬 실행 시
cd python-llm
pip install -r requirements.txt
```

### 2.5 vLLM 서버 (LLM 백엔드)

Python LLM 서버는 실제 추론을 외부 vLLM 서버(기본: `http://192.168.0.22:9000`)에 위임한다. vLLM 서버가 별도로 가동 중이어야 LLM 기능이 정상 동작한다.

```bash
# vLLM 서버 실행 예시 (GPU 서버에서)
python -m vllm.entrypoints.openai.api_server \
  --model Qwen/Qwen2.5-3B-Instruct \
  --port 9000
```

> **대체:** Ollama 백엔드 사용 시 `LLM_BACKEND=ollama`로 변경 가능.

## 4. Docker Compose 배포 (권장)

### 4.1 소스 코드 다운로드

```bash
# 배포 디렉토리 생성
```

### 4.2 SSL 인증서 준비

Nginx 컨테이너가 `./nginx/ssl/` 디렉토리의 인증서를 마운트한다.

```bash
# 자체 서명 인증서 생성 (개발/테스트)
mkdir -p nginx/ssl
openssl req -x509 -nodes -days 365 \
  -newkey rsa:2048 \
  -keyout nginx/ssl/server.key \
  -out nginx/ssl/server.crt \
  -subj "/CN=192.168.0.22"

# 운영 환경: Let's Encrypt 또는 기관 인증서를 nginx/ssl/ 에 배치
```

### 4.3 전체 서비스 기동

```bash
# .env 파일 설정 (3장 참조)
cp .env.example .env
vi .env

# 전체 빌드 및 기동
docker compose up -d --build

# 기동 상태 확인
docker compose ps
```

#### 기동 순서 (docker-compose.yml 의존성 자동 관리)

```
1. hms-db (MySQL)          ─ healthcheck 통과 대기
2. hms-redis (Redis)       ─ healthcheck 통과 대기
3. hms-chromadb (ChromaDB) ─ healthcheck 통과 대기
4. hms-python (LLM 서버)   ─ MySQL, ChromaDB 의존
5. hms-spring (Spring Boot) ─ MySQL, Redis, Python LLM 의존
6. hms-nginx (Nginx)       ─ Spring Boot 의존
```

### 4.4 서비스 관리

```bash
# 전체 중지
docker compose down

# 전체 재시작
docker compose restart

# 특정 서비스만 재빌드 & 재시작
docker compose up -d --build spring-app
docker compose up -d --build python-llm

# 로그 확인 (전체)
docker compose logs -f

# 특정 서비스 로그
docker compose logs -f spring-app
docker compose logs -f python-llm

# 헬스 체크
curl -k https://localhost/login || echo "서버 미응답"
```

## 5. 로컬 개발 환경 실행

개발 환경은 **H2 인메모리 DB**를 사용하며 MySQL/Redis 없이 단독 실행 가능하다.

### 5.1 Windows (PowerShell)

```powershell
# 1. .env 파일 생성
copy .env.example .env
# .env 에 CLAUDE_API_KEY, LLM_SERVICE_URL 등 필요한 값 기입

# 2. 개발 서버 실행
.\run-dev.ps1
```

`run-dev.ps1`이 수행하는 작업:

- `.env` 파일에서 환경변수 로드
- `SPRING_DATASOURCE_*` 환경변수 제거 (H2 사용을 위해)
- `--spring.profiles.active=dev`로 Spring Boot 기동

### 5.2 Linux / macOS

```bash
# .env 로드 후 dev 프로필로 실행
set -a && source .env && set +a
unset SPRING_DATASOURCE_URL SPRING_DATASOURCE_USERNAME SPRING_DATASOURCE_PASSWORD
./gradlew bootRun --args='--spring.profiles.active=dev'
```

## 6. 네이티브 빌드 & 실행 (Docker 미사용)

### 6.1 필수 소프트웨어

- JDK 21+
- MySQL 8.0+
- Redis 7.0+
- Node.js 20+
- Python 3.11+ (LLM 서버)

### 6.2 빌드

```bash
# 프론트엔드 빌드
npm ci && npm run build

# Gradle Wrapper 실행 권한 부여
chmod +x gradlew

# 빌드 (테스트 제외)
./gradlew clean bootJar -x test

# 빌드 결과 확인
ls -la build/libs/
# hms-0.0.1-SNAPSHOT.jar
```

### 6.3 실행

```bash
# 환경변수 설정 후 실행
java -Dspring.profiles.active=prod \
  -jar build/libs/hms-0.0.1-SNAPSHOT.jar

# 백그라운드 실행
nohup java -Dspring.profiles.active=prod \
  -jar build/libs/hms-0.0.1-SNAPSHOT.jar \
  > /opt/hms/logs/app.log 2>&1 &
```

### 6.4 Python LLM 서버 실행

```bash
cd python-llm
pip install -r requirements.txt

# 환경변수 설정
export MYSQL_USER=hms_admin
export MYSQL_PASSWORD=hms_password
export LLM_BACKEND=vllm
export VLLM_BASE_URL=http://192.168.0.22:9000

# 서버 실행
uvicorn app:app --host 0.0.0.0 --port 8000
```

### 6.5 systemd 서비스 등록 (운영)

```bash
sudo vi /etc/systemd/system/hms.service
```

```ini
[Unit]
Description=HMS - Hospital Management System
After=network.target mysql.service redis-server.service
Requires=mysql.service redis-server.service

[Service]
Type=simple
User=deploy
Group=deploy
WorkingDirectory=/opt/hms/hms

Environment="SPRING_DATASOURCE_URL=jdbc:mysql://localhost:3306/hms_db?useSSL=false&serverTimezone=Asia/Seoul&characterEncoding=UTF-8"
Environment="SPRING_DATASOURCE_USERNAME=hms_admin"
Environment="SPRING_DATASOURCE_PASSWORD=강력한_비밀번호_입력"
Environment="SPRING_DATA_REDIS_HOST=localhost"
Environment="SPRING_DATA_REDIS_PORT=6379"
Environment="LLM_SERVICE_URL=http://localhost:8000"
Environment="CLAUDE_API_KEY=sk-ant-api03-xxxxxxxxxxxx"
Environment="SERVER_PORT=8080"
Environment="LOG_PATH=/opt/hms/logs"

ExecStart=/usr/bin/java -Dspring.profiles.active=prod -jar /opt/hms/hms/build/libs/hms-0.0.1-SNAPSHOT.jar
ExecStop=/bin/kill -SIGTERM $MAINPID

Restart=on-failure
RestartSec=10

StandardOutput=append:/opt/hms/logs/app.log
StandardError=append:/opt/hms/logs/app-error.log

[Install]
WantedBy=multi-user.target
```

```bash
sudo mkdir -p /opt/hms/logs && sudo chown deploy:deploy /opt/hms/logs
sudo systemctl daemon-reload
sudo systemctl enable hms.service
sudo systemctl start hms.service
sudo systemctl status hms.service
```

## 7. DB 초기화

### 7.1 Docker Compose 사용 시

MySQL 컨테이너가 `MYSQL_DATABASE=hms_db`로 자동 생성하며, Spring Boot prod 프로필의 `ddl-auto=update`가 스키마를 자동 관리한다.

### 7.2 수동 초기화 (운영 환경 권장)

```sql
-- root로 접속
mysql -u root -p

-- 데이터베이스 생성 (Docker Compose 미사용 시)
CREATE DATABASE hms_db DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 전용 사용자 생성
CREATE USER 'hms_admin'@'%' IDENTIFIED BY '강력한_비밀번호_입력';
GRANT ALL PRIVILEGES ON hms_db.* TO 'hms_admin'@'%';
FLUSH PRIVILEGES;
```

### 7.3 테이블 구조 (JPA 엔티티 기반)

```
실행 순서 (외래 키 의존성):
1. department          -- 진료과 (독립)
2. staff               -- 직원 (FK: department)
3. doctor              -- 의사 (FK: staff, department)
4. patient             -- 환자 (독립)
5. reservation         -- 예약 (FK: patient, doctor, department)
6. treatment_record    -- 진료 기록 (FK: reservation, doctor)
7. hospital_rule       -- 병원 규칙 (독립)
```

## 8. Nginx & SSL 설정

### 8.1 기본 설정 (Docker Compose 내장)

`nginx/default.conf` 파일이 컨테이너에 마운트된다.

```nginx
server {
    listen 443 ssl;
    server_name 192.168.0.22;  # 실제 도메인/IP로 변경

    ssl_certificate     /etc/nginx/ssl/server.crt;
    ssl_certificate_key /etc/nginx/ssl/server.key;

    location / {
        proxy_pass http://spring-app:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto https;
    }
}

# HTTP → HTTPS 리다이렉트
server {
    listen 80;
    server_name 192.168.0.22;
    return 301 https://$host$request_uri;
}
```

### 8.2 SSL 인증서 (Let's Encrypt — 운영)

```bash
# Certbot 설치 (호스트에서)
sudo apt install -y certbot

# 인증서 발급
sudo certbot certonly --standalone -d example.com

# 발급된 인증서를 nginx/ssl/ 에 복사
sudo cp /etc/letsencrypt/live/example.com/fullchain.pem nginx/ssl/server.crt
sudo cp /etc/letsencrypt/live/example.com/privkey.pem nginx/ssl/server.key

# Nginx 컨테이너 재시작
docker compose restart nginx
```

### 9.1 배포 전 필수 확인 사항

| 순번 | 항목                                          | 확인 방법                                                                                 | 중요도   |
| ---- | --------------------------------------------- | ----------------------------------------------------------------------------------------- | -------- |
| 1    | **`.env` 파일 `.gitignore` 등록**             | `.gitignore` 파일 확인                                                                    | **필수** |

| 5    | **BCrypt 비밀번호 암호화**                    | 평문 비밀번호 DB 저장 여부 확인                                                           | **필수** |
| 6    | **SSL 인증서 적용**                           | HTTPS 접속 가능 확인                                                                      | 높음     |
| 7    | **Redis 외부 접근 차단**                      | Docker 네트워크 내부로 제한 또는 `bind 127.0.0.1`                                         | 높음     |
| 8    | **방화벽 규칙**                               | 필요 포트만 개방 (80, 443)                                                                | 높음     |
| 9    | **CORS 설정 확인**                            | `HMS_CORS_ALLOWED_ORIGINS`에 운영 도메인만 포함                                           | 높음     |
| 10   | **Spring Security CSRF 활성화**               | `SecurityConfig`에서 CSRF 비활성화하지 않았는지 확인                                      | 높음     |
| 11   | **ROLE 기반 URL 접근 제어**                   | 모든 관리자/직원 URL에 ROLE 제한 적용 확인                                                | 높음     |
| 12   | **JPA ddl-auto 설정**                         | 운영: `update`, 안정화 후 `validate` 또는 `none` 권장                                     | 높음     |
| 13   | **vLLM/Ollama 서버 접근 제한**                | 내부 네트워크에서만 접근 가능하도록 방화벽 설정                                           | 높음     |
| 14   | **ChromaDB 외부 접근 차단**                   | Docker 네트워크 내부로 제한                                                               | 보통     |

### 9.2 방화벽 설정 (ufw)

```bash
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow 22/tcp    # SSH
sudo ufw allow 80/tcp    # HTTP (→ HTTPS 리다이렉트)
sudo ufw allow 443/tcp   # HTTPS
sudo ufw enable
sudo ufw status verbose
```

#### 문제 1: MySQL healthcheck 실패로 서비스 기동 안 됨

```
dependency failed to start: container hms-db is unhealthy
```

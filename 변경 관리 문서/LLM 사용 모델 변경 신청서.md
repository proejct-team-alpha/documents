# LLM 사용 모델 변경 신청서

> **문서 유형:** 변경 관리 문서
> **작성일:** 2026년
> **연관 문서:** API 명세서 v5 (4. LLM 증상 추천 API, 8. LLM 규칙 챗봇 API)

---

## 1. 변경 개요

| 항목 | 내용 |
|------|------|
| **변경 유형** | LLM 백엔드 모델 교체 |
| **변경 전** | Claude AI (Anthropic API) |
| **변경 후** | Ollama 로컬 모델 |
| **영향 범위** | 증상 분석 추천, 규칙 챗봇 등 LLM 기반 AI 기능 전반 |

---

## 2. 변경 배경

### 2.1 기존 계획

프로젝트 초기부터 LLM 기반 AI 기능(증상 분석 추천, 규칙 챗봇 등)을 **Claude AI**를 활용하여 구현할 계획이었다. API 명세서 및 화면 정의서 등 기존 문서에는 Claude API 호출을 전제로 한 설계가 반영되어 있다.

### 2.2 변경 사유

| 사유 | 설명 |
|------|------|
| **토큰 비용 부담** | Claude API는 호출당 토큰 기반 과금이 발생한다. 증상 분석·규칙 챗봇 등 사용 빈도가 높을 경우 운영 비용이 지속적으로 증가하며, 소규모 병원 시스템에서는 부담이 될 수 있다. |
| **Ollama 및 내부 모델 사용 학습** | Ollama를 통해 로컬 LLM을 구축·운영함으로써, (1) 외부 API 의존도를 낮추고, (2) 로컬 환경에서의 모델 배포·튜닝 경험을 쌓으며, (3) 향후 자체 내부 모델 활용 역량을 확보하는 데 목적이 있다. |

---

## 3. 변경 내용

### 3.1 기술적 변경

| 구분 | 변경 전 | 변경 후 |
|------|---------|---------|
| **모델 소스** | Anthropic Claude API (클라우드) | Ollama 로컬 모델 |
| **연동 방식** | HTTP API 호출 (API Key 인증) | Ollama 로컬 서버 호출 (localhost 또는 내부 네트워크) |
| **비용** | 토큰당 과금 | 로컬 구축 비용 일회성 (서버·GPU 등) |
| **네트워크** | 외부 인터넷 필요 | 내부망만으로 동작 가능 |

### 3.2 영향 받는 기능

| 기능 | API | 설명 |
|------|-----|------|
| 증상 분석 추천 | `POST /llm/symptom/analyze` | 환자 증상 텍스트 → 진료과·의사·시간대 추천 |
| 규칙 챗봇 | `POST /llm/rules/ask` | 병원 규칙 관련 질의응답 |
| 챗봇 이력 조회 | `GET /llm/rules/history` | 챗봇 대화 이력 (LLM 호출 로직과는 무관, 데이터 조회만) |

---

## 4. 기대 효과

| 항목 | 내용 |
|------|------|
| **비용 절감** | 토큰 과금 제거로 장기 운영 비용 감소 |
| **데이터 보안** | 환자 증상·병원 규칙 등 민감 정보가 외부 API로 전송되지 않음 |
| **기술 역량** | Ollama 및 로컬 LLM 운영·튜닝 경험 축적 |
| **확장성** | 향후 자체 모델 또는 다른 오픈소스 모델로 교체 용이 |

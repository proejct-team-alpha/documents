# Git Reset

상황에 맞는 최적의 Git 되돌리기 방법을 안내하고 실행하는 스킬이다.

## 언제 사용하는가

- 변경 사항을 되돌리고 싶을 때
- "되돌려줘", "취소해줘", "커밋 취소", "변경 취소" 요청 시

## 동작 흐름

### Step 1: 현재 상황 파악

```bash
git status                    # 현재 작업 상태
git log --oneline -10         # 최근 커밋 히스토리
git stash list                # 스태시 목록 (있는 경우)
```

### Step 2: 사용자 의도 파악

사용자에게 무엇을 되돌리고 싶은지 질문한다:

```
어떤 것을 되돌리고 싶으신가요?

1. 아직 커밋하지 않은 파일 변경을 취소하고 싶다
2. 스테이징(git add)한 것을 취소하고 싶다
3. 방금 한 커밋을 취소하고 싶다
4. 이전 커밋 여러 개를 되돌리고 싶다
5. 변경 사항을 임시로 저장해두고 싶다
6. 임시 저장한 것을 복원하고 싶다
```

### Step 3: 상황별 최적 방법 실행

#### 상황 1: 파일 변경 취소 (언스테이지 상태)

특정 파일의 변경을 되돌린다.

```bash
# 특정 파일 되돌리기
git checkout -- {파일경로}

# 모든 변경 되돌리기 (주의: 복구 불가)
git checkout -- .
```

**설명:** 워킹 디렉토리의 변경을 마지막 커밋 상태로 되돌립니다. 되돌린 변경은 복구할 수 없습니다.

**위험도:** 높음 — 실행 전 반드시 확인

#### 상황 2: 스테이징 취소 (git add 취소)

스테이지에 올린 파일을 언스테이지로 되돌린다.

```bash
# 특정 파일 언스테이지
git reset HEAD {파일경로}

# 모든 파일 언스테이지
git reset HEAD
```

**설명:** 파일 내용은 그대로 유지하고, 스테이징만 취소합니다. 코드는 변경되지 않습니다.

**위험도:** 낮음 — 안전한 작업

#### 상황 3: 마지막 커밋 취소

```bash
# 커밋만 취소, 변경 내용은 스테이지에 유지 (추천)
git reset --soft HEAD~1

# 커밋 취소, 변경 내용은 언스테이지로 유지
git reset --mixed HEAD~1

# 커밋 취소, 변경 내용도 삭제 (주의: 복구 불가)
git reset --hard HEAD~1
```

**각 옵션 차이:**

| 옵션 | 커밋 | 스테이지 | 파일 | 위험도 |
|------|------|----------|------|--------|
| `--soft` | 취소 | 유지 | 유지 | 낮음 |
| `--mixed` | 취소 | 취소 | 유지 | 낮음 |
| `--hard` | 취소 | 취소 | 삭제 | **높음** |

**추천:** 먼저 `--soft`를 사용하고, 필요하면 단계적으로 진행

#### 상황 4: 이전 커밋 되돌리기 (이력 유지)

이미 push한 커밋이거나 이력을 보존하고 싶을 때 사용한다.

```bash
# 특정 커밋의 변경을 반대로 적용하는 새 커밋 생성
git revert {커밋해시}

# 여러 커밋 되돌리기
git revert {시작해시}..{끝해시}
```

**설명:** 원래 커밋은 이력에 남고, 되돌리는 내용이 새 커밋으로 추가됩니다. push된 커밋을 되돌릴 때 안전합니다.

**위험도:** 낮음 — 이력이 보존되므로 안전

**reset vs revert 선택 기준:**
- 아직 push하지 않은 로컬 커밋 → `reset`
- 이미 push한 커밋 → `revert` (force push 방지)

#### 상황 5: 임시 저장 (stash)

```bash
# 현재 변경 사항을 임시 저장
git stash

# 메시지와 함께 저장
git stash save "작업 중인 게시글 기능"

# 스태시 목록 확인
git stash list
```

**설명:** 커밋하지 않은 변경 사항을 임시로 저장하고 워킹 디렉토리를 깨끗하게 만듭니다. 브랜치를 전환해야 할 때 유용합니다.

#### 상황 6: 임시 저장 복원 (stash pop/apply)

```bash
# 가장 최근 스태시 복원 후 삭제
git stash pop

# 특정 스태시 복원 후 삭제
git stash pop stash@{번호}

# 복원만 하고 스태시는 유지
git stash apply stash@{번호}

# 스태시 삭제
git stash drop stash@{번호}
```

## 위험 명령 안전장치

`--hard`, `--force` 같은 위험 명령 실행 전 반드시 다음을 수행한다:

1. **현재 상태 백업 안내**: `git stash` 또는 `git branch backup-{날짜}`
2. **영향 범위 설명**: 어떤 파일/커밋이 삭제되는지 구체적으로 나열
3. **사용자 최종 확인**: "정말로 실행하시겠습니까? 이 작업은 복구할 수 없습니다."

## 주의사항

- `--hard` reset은 사용자가 명시적으로 요청한 경우에만 실행한다
- push된 커밋에 대해 `reset`을 사용하면 force push가 필요하다는 점을 반드시 경고한다
- 되돌리기 전 항상 `git status`와 `git log`로 현재 상태를 먼저 보여준다
- 의도를 명확히 파악하지 못했으면 가장 안전한 옵션부터 제안한다

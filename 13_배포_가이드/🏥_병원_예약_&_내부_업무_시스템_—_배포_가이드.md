### 4.3 systemd 서비스 등록 (운영 권장)

```bash
sudo vi /etc/systemd/system/hospital.service
```

```ini
[Unit]
Description=Hospital Reservation System
After=network.target mysql.service redis-server.service
Requires=mysql.service redis-server.service

[Service]
Type=simple
User=deploy
Group=deploy
WorkingDirectory=/opt/hospital/hospital-reservation

# 환경변수 설정
Environment="CLAUDE_API_KEY=sk-ant-api03-xxxxxxxxxxxx"
Environment="SPRING_DATASOURCE_URL=jdbc:mysql://localhost:3306/hospital_db?useSSL=false&serverTimezone=Asia/Seoul&characterEncoding=UTF-8"
Environment="SPRING_DATASOURCE_USERNAME=hospital_user"
Environment="SPRING_DATASOURCE_PASSWORD=강력한_비밀번호_입력"
Environment="SPRING_DATA_REDIS_HOST=localhost"
Environment="SPRING_DATA_REDIS_PORT=6379"
Environment="SERVER_PORT=8080"

ExecStart=/usr/bin/java -jar /opt/hospital/hospital-reservation/build/libs/hospital-reservation-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod
ExecStop=/bin/kill -SIGTERM $MAINPID

# 자동 재시작
Restart=on-failure
RestartSec=10

# 로그
StandardOutput=append:/opt/hospital/logs/app.log
StandardError=append:/opt/hospital/logs/app-error.log

[Install]
WantedBy=multi-user.target
```

```bash

sudo chown deploy:deploy /opt/hospital/logs

# 서비스 등록 및 시작
sudo systemctl daemon-reload
sudo systemctl enable hospital.service
sudo systemctl start hospital.service

# 상태 확인
sudo systemctl status hospital.service
```

## 5. DB 초기화

### 5.1 DDL 스크립트 실행 순서

> JPA `ddl-auto=update` 사용 시 자동 생성되지만, 운영 환경에서는 수동 DDL 관리를 권장한다.

아래 순서대로 실행해야 외래 키 제약 조건 오류가 발생하지 않는다.

```
실행 순서:
1. department.sql         -- 진료과 (독립 테이블)
2. staff.sql              -- 직원 (독립 테이블)
3. doctor.sql             -- 의사 (FK: department)
4. patient.sql            -- 환자 (독립 테이블)
5. reservation.sql        -- 예약 (FK: patient, doctor, department)
6. treatment_record.sql   -- 진료 기록 (FK: reservation, doctor)
7. hospital_rule.sql      -- 병원 규칙 (독립 테이블)
8. llm_recommendation.sql -- LLM 추천 (독립 테이블)
```

```bash
# DDL 일괄 실행
mysql -u hospital_user -p hospital_db < sql/schema.sql
```

## 6. Nginx 리버스 프록시 설정 (선택)

### 6.1 설정 파일 작성

```bash
sudo vi /etc/nginx/sites-available/hospital
```

```nginx
server {
    listen 80;
    server_name example.com;  # 실제 도메인으로 변경

    # HTTPS 리다이렉트 (SSL 설정 시)
    # return 301 https://$host$request_uri;

    # 리버스 프록시
    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # 타임아웃 설정 (LLM API 응답 대기 고려)
        proxy_connect_timeout 10s;
        proxy_read_timeout 30s;
        proxy_send_timeout 10s;
    }

    # 정적 리소스 캐싱 (선택)
    location /static/ {
        proxy_pass http://127.0.0.1:8080/static/;
        expires 7d;
        add_header Cache-Control "public, immutable";
    }

    # 파일 크기 제한
    client_max_body_size 10M;
}
```

```bash
# 심볼릭 링크 생성
sudo ln -s /etc/nginx/sites-available/hospital /etc/nginx/sites-enabled/

# 기본 설정 제거
sudo rm /etc/nginx/sites-enabled/default

# 설정 검증
sudo nginx -t

# 적용
sudo systemctl reload nginx
```

| 10   | **ROLE 기반 URL 접근 제어**              | 모든 관리자/직원 URL에 ROLE 제한 적용 확인                       | 높음     |
| 11   | **BCrypt 비밀번호 암호화**               | 평문 비밀번호 DB 저장 여부 확인                                  | **필수** |
| 12   | **JPA ddl-auto 설정**                    | 운영 환경에서 `validate` 또는 `none` 사용                        | 높음     |

### 7.2 방화벽 설정 (ufw)

```bash
# 기본 정책: 들어오는 트래픽 차단, 나가는 트래픽 허용
sudo ufw default deny incoming
sudo ufw default allow outgoing

# SSH 허용
sudo ufw allow 22/tcp

# HTTP / HTTPS 허용
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# MySQL 외부 접근 차단 (기본적으로 차단됨, 확인용)
# sudo ufw deny 3306/tcp

# Redis 외부 접근 차단
# sudo ufw deny 6379/tcp

# 방화벽 활성화
sudo ufw enable
sudo ufw status verbose
```

## 8. 트러블슈팅

### 8.1 자주 발생하는 문제 및 해결 방법

#### 문제 1: 애플리케이션 실행 시 MySQL 연결 실패

```
com.mysql.cj.jdbc.exceptions.CommunicationsException:
```

#### 문제 8: 한글 깨짐 (MySQL UTF-8)

**원인:** MySQL 기본 문자셋이 utf8mb4가 아님

**해결:**

```bash
# MySQL 문자셋 확인
mysql -u root -p -e "SHOW VARIABLES LIKE 'character_set%';"

# my.cnf 설정
sudo vi /etc/mysql/mysql.conf.d/mysqld.cnf
```

```ini
[mysqld]
character-set-server = utf8mb4
collation-server = utf8mb4_unicode_ci

[client]
default-character-set = utf8mb4
```

```bash
sudo systemctl restart mysql
```

JDBC URL에도 문자셋을 명시해야 한다.

```
jdbc:mysql://localhost:3306/hospital_db?useSSL=false&serverTimezone=Asia/Seoul&characterEncoding=UTF-8
```

#### 문제 9: Spring Security 로그인 후 무한 리다이렉트

**원인:** 로그인 성공 후 리다이렉트 URL이 다시 로그인 페이지를 가리킴

**해결:**

- `SecurityConfig`에서 `defaultSuccessUrl`이 올바르게 설정되었는지 확인한다.
- 로그인 페이지 URL이 `.permitAll()`로 설정되었는지 확인한다.

```java
// SecurityConfig.java
http.formLogin(form -> form
    .loginPage("/login")
    .defaultSuccessUrl("/dashboard", true)  // true: 항상 이 URL로 이동
    .permitAll()
);
```
